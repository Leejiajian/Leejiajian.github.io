TIME_WAIT 状态

  • 可靠地终止TCP连接
  • 保证迟来的TCP报文有足够的时间被识别并丢弃

复位报文段

下面是几种会发送复位报文段的场景

  • 访问不存在的端口
  • 异常终止连接
    发送给对方一个复位报文段，发送端所有排队等待发送的数据都将被丢弃
  • 处理半打开状态：
    当往半打开状态的连接中写入数据，则对方会回应一个复位报文段

tcp 拥塞控制

  1. 慢启动
  2. 拥塞避免
  3. 快速重传
  4. 快速恢复

拥塞控制控制的是发送端向网络一次连续写入(收到其中第一个数据的确认之前)的数据量，称为SWND(发送窗口)。

chapter 5

主机字节序和网络字节序

大端字节序：高位字节存储在内存的低地址处，低位存储在内存的高地址处。

小端字节序：高位字节存储在内存的高地址处，低位存储在内存的低地址处。

主机字节序：小端字节序

网络字节序：大端字节序

java 虚拟机采用大端字节序

Linux 字节序转换的函数

#include<netinet/in.h>
unsigned long int htonl(unsigned long int hostlong);
unsigned short int htons(unsigned short int hostshort);
unsigned long int ntohl(unsigned long int netlong);
unsigned short int ntohs(unsigned short int netshort);

htonl 表示 “host to network long”

长整型通常用来转换IP地址，短整型用来转换端口号。

通用 socket 地址

socket网络接口中表示socket地址的是结构体sockaddr,其定义如下：

#include<bits/socket.h>
struct sockaddr {
    sa_family_t sa_family;
    char sa_data[14];
};
协议族 地址族 描述
PF_UNIX AF_UNIX UNIX本地域协议族
PF_INET AF_INET TCP/IPv4协议族
PF_INET6 AF_INET6 TCP/IPv6议族
协议族 地址值含义和长度
PF_UNIX 文件的路径名，长度可达108字节
PF_INET 16bit的端口号和32bit IPv4地址，共6字节
PF_INET6 16bit端口号，32bit流标识，128bit IPv6地址，32bit范围ID,共26字节

UNIX 本地域协议族

#include<sys/un.h>
struct sockaddr_un{
    sa_family_t sin_family; // 地址族
    char sun_path[108];  // 文件路径名
};

TCP/IP 协议族

// IPv4 地址
struct sockaddr_in
{
    sa_family_t sin_family;/*地址族AF_INET*/
    u_int16_t sin_port;/*端口号，要用网络字节序表示*/
    struct in_addr sin_addr;/*IPv4地址结构体，见下面*/
};
struct in_addr
{
    u_int32_t s_addr;/*IPv4地址，要用网络字节序表示*/
};

//IPv6地址表示
struct sockaddr_in6
{
    sa_family_t sin6_family;/*地址族：AF_INET6*/
    u_int16_t sin6_port;/*端口号，要用网络字节序表示*/
    u_int32_t sin6_flowinfo;/*流信息，应设置为0*/
    struct in6_addr sin6_addr;/*IPv6地址结构体，见下面*/
    u_int32_t sin6_scope_id;/*scope ID，尚处于实验阶段*/
};
struct in6_addr
{
    unsigned char sa_addr[16];/*IPv6地址，要用网络字节序表示*/
};

所有专用的socket地址类型的变量在使用过程中都需要转化为sockaddr(强制类型转换即可)。

IP地址转换函数

#include＜arpa/inet.h＞
// 将用点分十进制字符串表示的IPv4地址转化为用网络字节序整数表示的IPv4地址
in_addr_t inet_addr(const char*strptr);
int inet_aton(const char*cp,struct in_addr*inp);
// 将用网络字节序整数表示的IPv4地址转化为用点分十进制字符串表示的IPv4地址
char*inet_ntoa(struct in_addr in);

下面的函数也能完成和前面的三个函数同样的功能，并且同时适用于IPv4和IPv6

#include<arpa/inet.h>
int inet_pton(int af,const char*src,void*dst);
const char*inet_ntop(int af,const void*src,char*dst,socklen_t cnt);

将用字符串表示的IP地址src（用点分十进制字符串表示的IPv4地址或用十六进制字符串表示的IPv6地址）转换成用网络字节序整数表示的IP地址，并把转换结果存储于dst指向的内存中.

af : 指定地址族，可以是AF_INET 或 AF_INET6

inet_ntop函数与inet_pton函数进行相反的操作，最后的参数指定目标存储单元的大小。可通过下面的宏指定大小：

#include＜netinet/in.h＞
#define INET_ADDRSTRLEN 16
#define INET6_ADDRSTRLEN 46

inet_ntop成功时返回目标存储单元的地址，失败则返回NULL并设置errno。

创建 socket

#include<sys/types.h>
#include<sys/socket.h>
int socket(int domain, int type, int protocol);

domain: 明确使用哪个底层协议族。对于TCP/IP协议族，使用PF_INET或者PF_INET6。

type: 指定服务类型。主要有SOCK_STREAM(流服务)和SOCK_UGRAM(数据包服务)；SOCK_NONBLOCK:将新创建的socket设置为非阻塞的；SOCK_CLOEXEC:用fork调用创建子进程时在子进程中关闭该socket.

protocol: 设置为0，表示使用默认协议.

返回值：调用成功返回一个socket文件描述符，失败则返回-1并设置errno.

命名 socket

  • 创建socket时，指定了地址族，但未指定使用该地址族的哪个具体socket地址。将socket与socket地址绑定称为命名socket。

#include＜sys/types.h＞
#include＜sys/socket.h＞
int bind(int sockfd,const struct sockaddr*my_addr,socklen_t addrlen);

sockfd: socket文件描述符

my_addr: socket地址

addlen: 该socket地址的长度

返回值：成功时返回0， 失败返回1，并设置errno。

创建监听socket

  • 创建监听队列存放待处理的客户连接

#include<sys/socket.h>
int listen(int sockfd, int backlog);

backlog: 内核监听队列的最大长度

返回值：成功时返回0，失败时返回-1并设置errno.

接收连接

#include<sys/types.h>
#include<sys/socket.h>
int accept(int sockfd, struct sockaddr *addr, socklen_t *addrlen);

sockfd: 执行过listen系统调用的监听socket。

addr:用来获取被接受连接的远端socket地址

addrlen:该socket地址的长度

返回值：成功时返回一个新的连接socket, 该socket唯一标识了被接受的这个连接，服务器可通过读写该socket来与被接受连接对应的客户端通信。失败时返回-1,并设置errno.

服务端

#include<sys/types.h>
#include<sys/socket.h>
int connect(int sockfd, const struct sockaddr *serv_addr, socklen_t addrlen);

serv_addr: 服务器监听的socket地址

addrlen: 该地址的长度

返回值：成功时返回0， 失败时返回-1并设置errno

#include<unistd.h>
int close(int fd);

fd是待关闭的socket。close将fd的引用计数减一，引用计数等于0时，关闭连接。

#include<sys/socket.h>
int shutdown(int sockfd, int howto);

howto 决定shutdown的行为。

SHUT_RD:关闭socket上读的这一半。不能再对socket文件描述符执行读操作，并且该socket接收缓冲区中的数据都被丢弃。

SHUT_WR:关闭socket上写的这一半。sockfd的发送缓冲区数据会在真正关闭连接之前全部发送出去，不可再对该socket文件描述符执行写操作。

SHUT_RDWR: 同时关闭sockfd上的读和写。

数据读写

TCP 数据读写

#include<sys/types.h>
#include<sys/socket.h>
ssize_t recv(int sockfd, void *buf, size_t len, int flags);
ssize_t send(int sockfd, const void *buf, size_t len, int flags);

recv: flags通常设置为0

都返回实际写入的长度

UDP 数据读写

#include<sys/types.h>
#include<sys/socket.h>
ssize_t recvfrom(int sockfd,void*buf,size_t len,int flags,struct sockaddr *src_addr,socklen_t *addrlen);
ssize_t sendto(int sockfd,const void*buf,size_t len,int flags,const struct sockaddr *dest_addr,socklen_t addrlen);

通用数据读写函数

#include<sys/socket.h>
ssize_t recvmsg(int sockfd,struct msghdr*msg,int flags);
ssize_t sendmsg(int sockfd,struct msghdr*msg,int flags);


struct msghdr
{
    void* msg_name;/*通信对方socket地址*/
    socklen_t msg_namelen;/*socket地址的长度*/
    struct iovec* msg_iov;/*分散的内存块，见后文*/
    int msg_iovlen;/*分散内存块的数量*/
    void* msg_control;/*指向辅助数据的起始位置*/
    socklen_t msg_controllen;/*辅助数据的大小*/
    int msg_flags;/*复制函数中的flags参数，并在调用过程中更新*/
};


struct iovec {
    void *iov_base; // 内存起始地址
    size_t iov_len; // 这块内存的长度
};
